# Resources and References

* **[Nine Men's Morris - Wikipedia](https://en.wikipedia.org/wiki/Nine_men%27s_morris)**
    * Used as the main reference for the game's exact rules, phases, and board layout.

* **[Ralph Gasser: Solving Nine Men's Morris (PDF)](https://www.cs.brandeis.edu/~storer/JimPuzzles/GAMES/NineMensMorris/INFO/GasserArticle.pdf)**
    *  Provided theoretical background on the complexity of the game.

* **[MCTS Guide - AI Boson](https://ai-boson.github.io/mcts/)**
    * Helped to understand and structure the four core phases of the MCTS algorithm. 

* **[The Animated Monte Carlo Tree Search (MCTS) - Medium](https://medium.com/data-science/the-animated-monte-carlo-tree-search-mcts-c05bb48b018c)**
    * Provided visual animations explaining how the search tree expands over time.

* **[Monte Carlo Tree Search (MCTS) in Machine Learning - GeeksforGeeks](https://www.geeksforgeeks.org/machine-learning/monte-carlo-tree-search-mcts-in-machine-learning/)**
    * Used for technical implementation details and understanding the math behind node exploration. It helped translate conceptual MCTS logic into actual Python dictionaries.

* **[AI in Nine Men's Morris Game - Jitpaul's Blog](https://jitpaul.blog/2017/07/18/ai-in-nine-men-s-morris-game/)**
    * Provided inspiration for domain-specific AI logic, such as how to evaluate board positions and value mills. It explained why the AI needed heuristic overrides like our attack and defense instincts.

* **[What are some efficient alternatives to a Monte Carlo Tree Search? - Reddit](https://www.reddit.com/r/learnprogramming/comments/10ew6zx/what_are_some_efficient_alternatives_to_a_monte/)**
    * Prompted us to choose MCTS over Minimax for this project.

* **[ArXiv Paper on Search Algorithms (1408.0032v3)](https://arxiv.org/html/1408.0032v3)**
    * Used to understand advanced tree search optimizations. It also provided context on reinforcement learning concepts within board games.

* **[Google Gemini](https://gemini.google.com)**
